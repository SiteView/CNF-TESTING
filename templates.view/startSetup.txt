 <hr><h3><a name="getstart"></a>Tips to Get Started</h3><P>SiteView is accessed through a web browser by entering the
 IP address and port number of the SiteView service. The SiteView
 <a href="/SiteView/docs/GetStart.htm" target="Help">main page</a> is your entry point to SiteView.
 SiteView <a href="/SiteView/docs/Monitors.htm" target="Help">monitors</a> are <b>agentless</b>, automated tests of services
 or systems in your Web environment. Monitors are held in
 <a href="/SiteView/docs/Group.htm" target="Help">groups</a> that can be organized according to importance,
 location, type of system, or other criteria. Click on the name of a group to add or edit monitors and to
 drill down into <a href="/SiteView/docs/Group.htm" target="Help">monitoring data</a>. Use the navigation
 bar to access <a href="/SiteView/docs/Alert.htm" target="Help">alert</a> settings, performance
 <a href="/SiteView/docs/HReports.htm" target="Help">reports</a>, or view the
 <a href="/SiteView/UserGuide.htm">on-line help</a>. Click on the
 <a href="/SiteView/docs/GenPref.htm" target="Help">Preferences</a> link to access a large number of
 configuration options.</p> 
 <p>To get started with SiteView we recommend you do the following:</p>
 <ol><li>Decide what web systems, servers, and URL's you want to monitor</li>
 <li>Decide how the monitoring can be grouped and create
 new SiteView  <a href="/SiteView/docs/Group.htm" target="Help">monitor groups</a> to organize the monitors</li>
 <li>After creating monitor groups, click on the name of the group to add
 <a href="/SiteView/docs/Monitors.htm" target="Help">monitors</a> to that group.
 To set up monitors you will need to know:
 <ol type="a"><li>How to connect to the system or server you want to monitor</li>
 <li>What you want to monitor on that system</li></ol> 
 Once monitors are set up, SiteView will begin reporting results
 within minutes.</li>
 <li>After creating monitors, set up automated
 <a href="/SiteView/docs/Alert.htm">alerts</a> using the media that best suit your organization.
 For e-mail alerts, enter the mail server address that SiteView should use below.
</li>
 <li>Later, you can set up scheduled <a href="/SiteView/docs/HReports.htm" target="Help">reports</a> for key
 monitors to review system availability over time</li></ol> 
 <p>Start with a few monitors and systems. Once you become
 comfortable with the product, you can easily set up more monitors,
 <a href="/SiteView/docs/Monitors.htm#remote" target="Help">monitor remote systems</a>, adjust alerts,
 and make other changes to best suit your needs.<p></p>
